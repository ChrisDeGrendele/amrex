50 /*
51  A Collection of Fortran Array-like Objects
52 
53 
54  The FabArray<FAB> class implements a collection (stored as an array) of
55  Fortran array-like objects. The parameterized type FAB is intended to be
56  any class derived from BaseFab<T>. For example, FAB may be a BaseFab of
57  integers, so we could write:
58 
59  FabArray<BaseFab<int> > int_fabs;
60 
61  Then int_fabs is a FabArray that can hold a collection of BaseFab<int>
62  objects.
63 
64  FabArray is not just a general container class for Fortran arrays. It is
65  intended to hold "grid" data for use in finite difference calculations in
66  which the data is defined on a union of (usually disjoint) rectangular
67  regions embedded in a uniform index space. This region, called the valid
68  region, is represented by a BoxArray. For the purposes of this discussion,
69  the Kth Box in the BoxArray represents the interior region of the Kth grid.
70 
71  Since the intent is to be used with finite difference calculations a
72  FabArray also includes the notion of a boundary region for each grid. The
73  boundary region is specified by the ngrow parameter which tells the FabArray
74  to allocate each FAB to be ngrow cells larger in all directions than the
75  underlying Box. The larger region covered by the union of all the FABs is
76  called the region of definition. The underlying notion is that the valid
77  region contains the grid interior data and the region of definition includes
78  the interior region plus the boundary areas.
79 
80  Operations are available to copy data from the valid regions into these
81  boundary areas where the two overlap. The number of components, that is,
82  the number of values that can be stored in each cell of a FAB, is either
83  given as an argument to the constructor or is inherent in the definition of
84  the underlying FAB. Each FAB in the FabArray will have the same number of
85  components.
86 
87  In summary, a FabArray is an array of FABs. The Kth element contains a FAB
88  that holds the data for the Kth grid, a Box that defines the valid region
89  of the Kth grid.
90 
91  A typical use for a FabArray would be to hold the solution vector or
92  right-hand-side when solving a linear system of equations on a union of
93  rectangular grids. The copy operations would be used to copy data from the
94  valid regions of neighboring grids into the boundary regions after each
95  relaxation step of the iterative method. If a multigrid method is used, a
96  FabArray could be used to hold the data at each level in the multigrid
97  hierarchy.
98 
99  This class is a concrete class not a polymorphic one.
100 
101  This class does NOT provide a copy constructor or assignment operator.
102 */
